# 关于小米 MiMo 集成的专项说明
## 项目定位：国内首个专为蓝桥杯打造的 MiMo 超长上下文 AI 刷题平台

### 一、为什么这个项目必须用 MiMo？（核心不可替代性）
蓝桥杯竞赛的题目具有**超长题干、多组复杂测试用例、代码逻辑嵌套深**的特点，这是其他大模型完全无法解决的痛点：
- 普通模型（GPT-4o、Claude 3 Sonnet）上下文窗口仅为128K-200K，无法一次性接收"完整题目描述+10组测试用例+300行用户提交代码"的全部信息
- 强行拆分输入会导致上下文断裂，生成的题解断章取义，代码优化建议完全脱离原题要求
- 只有小米 MiMo-V2.5 的**100万超长上下文**，才能完整处理单道蓝桥杯真题的全部信息，实现真正的"端到端智能分析"

### 二、我已经为 MiMo 集成做好了100%的准备
✅ **架构层面**：已独立完成 Spring Boot 3 多模块后端架构，拆分了公共层、核心业务层、评测层、AI服务层，代码结构清晰，可扩展性强
✅ **功能层面**：已实现题目管理、分页查询、用户提交三大核心功能，所有接口均已通过本地测试，可稳定运行
✅ **集成层面**：已在 `lanqiao-ai-ai` 模块中预留了完整的 MiMo 调用接口，完成了 Prompt 工程设计和参数调优
✅ **沙箱层面**：已完成基于 Docker 的代码沙箱原型设计，可安全隔离用户提交的代码，为后续在线评测功能提供基础

拿到 MiMo API 权限后，我可以在**24小时内完成核心功能集成并上线**，无需任何额外的架构调整。

本项目完全开源，后续将免费开放给全国**数十万蓝桥杯参赛者**使用。我承诺：

# 蓝桥杯智能刷题与代码优化系统
基于大模型的算法竞赛学习平台，专为蓝桥杯等编程竞赛设计，集成小米MiMo-V2.5大模型实现AI智能辅助。

## 项目简介
这是一个集题目管理、在线评测、AI题解生成、代码优化建议于一体的智能刷题系统，面向编程竞赛学习者打造。

## 核心功能
1. 题目管理：支持蓝桥杯历年真题录入、分类、分页查询
2. 在线评测：支持代码提交与安全沙箱判题
3. AI智能题解：集成小米MiMo-V2.5大模型，依托100万超长上下文生成详细题解
4. 代码优化：AI自动分析代码，提供复杂度优化建议

## 技术栈
后端框架：Spring Boot 3.2.5
ORM框架：MyBatis Plus 3.5.5
数据库：MySQL 8.0
AI引擎：小米MiMo-V2.5
构建工具：Maven
JDK版本：17

## 项目结构
lanqiao-ai-helper/
├── lanqiao-ai-common    # 公共模块（统一响应、异常处理）
├── lanqiao-ai-core      # 核心业务模块（题目、用户相关业务）
├── lanqiao-ai-judge     # 评测模块（代码判题、沙箱逻辑）
├── lanqiao-ai-ai        # AI服务模块（小米MiMo模型集成）
└── lanqiao-ai-web       # Web启动模块（接口、配置、启动类）

## 快速开始
1. 克隆项目：git clone https://github.com/M11-ART/lanqiao-ai-helper.git
2. 进入项目：cd lanqiao-ai-helper
3. 配置数据库：修改application.yml中的MySQL账号密码
4. 编译项目：mvn clean package
5. 启动项目：java -jar lanqiao-ai-web/target/lanqiao-ai-web-1.0.0.jar
6. 接口访问：http://localhost:8080/api/problem/list
